# athlete_injury_risk_assesment
What is This?
The last 4 weeks through March and April I attended multiple meetings for the Biotech Club over creating a machine learning model to determine if an athlete is injured or not. We explored the libraries in Python and how they read the data before we ultimately trained 2 models which were the decision tree model and the logistic regression model.

What did I learn?
We imported multiple libraries with functions that I learned to use in order to read the data in a provided csv file. The file contained different categories which the machine learned that some impact the injury risk assesment more than others. In the creation of the model I also learned various actions like cleaning the data and making it usable that has to be done before it is trained.

- Explored and analyzed a structured athlete dataset to understand key variables related to injury risk.
- Verified data quality by checking for missing values and confirming dataset completeness before modeling.
- Converted categorical features such as gender and position into numerical form using one-hot encoding.
- Removed non-predictive identifiers to ensure the model focuses only on relevant performance and health metrics.
- Separated input features and target labels to properly structure the dataset for supervised learning.
- Standardized numerical variables using feature scaling to improve model performance and consistency.
- Analyzed correlations between features and injury outcomes to identify the most influential risk factors.
- Split the dataset into training and testing sets with stratification to maintain class balance.
- Trained and compared Logistic Regression and Decision Tree models to evaluate predictive performance.
- Built an interactive Streamlit application that replicates preprocessing steps and generates real-time predictions.
- Implemented input validation in the app to ensure user data stays within realistic and trained ranges.

How do you run it?
I have added it in a way so that it is linked with Google Colab suggesting that it should be runable if you simply click the link "open in Colab". As well there is going to be a link to directly access the trained model on a seperate website so a user can input data and see the output in a clean fashion.
